# RoadGuard Backend

RoadGuard Backend is a FastAPI service for ingesting, storing, and serving road hazard intelligence generated by the RoadGuard mobile app.

## Features

- FastAPI application startup with database bootstrap
- PostgreSQL connectivity through SQLAlchemy 2.x
- CORS enabled for local mobile development
- Health endpoint at `GET /health`
- Hazard ingestion and nearby hazard lookup
- Device registration endpoint
- Model registry creation, listing, and activation endpoints
- Basic centralized error handling

## Requirements

- Python 3.11+
- PostgreSQL 14+

## Setup

1. Create and activate a virtual environment.
2. Install dependencies.
3. Copy `.env.example` to `.env`.
4. Create the PostgreSQL database.
5. Start the API.

```bash
cd roadguard_backend
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
uvicorn app.main:app --reload
```

## Environment Variables

- `ROADGUARD_DATABASE_URL`

## Endpoints

- `GET /health`
- `POST /api/hazards`
- `GET /api/hazards/nearby`
- `GET /api/hazards/{hazard_id}`
- `POST /api/hazards/{hazard_id}/verify`
- `POST /api/hazards/{hazard_id}/false-positive`
- `POST /api/devices/register`
- `GET /api/devices/{device_id}`
- `GET /api/models`
- `GET /api/models/active`
- `POST /api/models`
- `PUT /api/models/{model_id}/activate`

## API Examples

Create a hazard:

```bash
curl -X POST http://127.0.0.1:8000/api/hazards \
  -H "Content-Type: application/json" \
  -d '{
    "device_id": "android-pixel-8a",
    "user_id": "user_123",
    "hazard_type": "pothole",
    "risk_level": "high",
    "latitude": 40.7128,
    "longitude": -74.0060,
    "confidence": 0.93,
    "estimated_distance_meters": 18.2,
    "estimated_width_meters": 0.8,
    "estimated_depth_meters": 0.15,
    "image_url": "https://cdn.example.com/hazards/abc123.jpg",
    "detected_at": "2026-06-26T07:30:00Z"
  }'
```

Fetch nearby hazards:

```bash
curl "http://127.0.0.1:8000/api/hazards/nearby?latitude=40.7128&longitude=-74.0060&radius_meters=250"
```

Verify a hazard:

```bash
curl -X POST http://127.0.0.1:8000/api/hazards/<hazard_id>/verify
```

Report a false positive:

```bash
curl -X POST http://127.0.0.1:8000/api/hazards/<hazard_id>/false-positive
```

Create a model version:

```bash
curl -X POST http://127.0.0.1:8000/api/models \
  -H "Content-Type: application/json" \
  -d '{
    "id": "rg-objdet-android-v1",
    "model_name": "roadguard_object_detection",
    "model_type": "object_detection",
    "version": "1.0.0",
    "platform": "android",
    "download_url": "https://cdn.example.com/models/roadguard_object_detection_android_v1.tflite",
    "checksum": "sha256:abcd1234",
    "input_width": 320,
    "input_height": 320,
    "labels_url": "https://cdn.example.com/models/labels.txt",
    "is_active": true
  }'
```

Fetch active model versions:

```bash
curl "http://127.0.0.1:8000/api/models/active?platform=android&model_type=object_detection"
```

Activate a model version:

```bash
curl -X PUT http://127.0.0.1:8000/api/models/rg-objdet-android-v1/activate
```

## Notes

- Database tables are created automatically at startup for local development.
- Authentication is intentionally not implemented yet.
- Backend changes under `roadguard_backend/` can be used to trigger the Cloud Run deployment workflow from GitHub Actions.
- For production, add Alembic migrations, structured logging, rate limiting, and authentication before deployment.
